

На базе Django + Django Rest Framework организовать хранение и получение через API данных пользователя:
- Сон
- Шаги
- Гео данные (присутствие пользователя в гео-точке)

Все модели хранят данные в виде промежутков:
- Сон (время начала, время конца)
- Шаги (время начала, время конца, количество шагов за промежуток)
- Гео (время начала, время конца, координаты)

Пример: сон "с 22 вечера до 7 утра" хранится не единым интервалом (с 22 до 7), а отдельными промежутками разной длительности (так устройства передают). Промежутки могут быть длиной до суток, но обычно - длиной от единиц секунд до пары часов.

API возвращает список объектов из таблицы-модели. Поля JSON объекта - id, time_start, time_end, ничего/value/latitude, longitude. API принимает два параметра time_start и time_end и возращает элементы, которые имеют *время начала* в промежутке от time_start до time_end.

В результате должно получится несколько моделей и пачка views для получения данных пользователя за промежуток времени (указанный в запросе) + набор тестов.

*оценка на реализацию - 2 часа с перекурами.
